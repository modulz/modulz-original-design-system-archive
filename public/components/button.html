Button

Buttons allow users to take significant action—submit a form, follow a user, create a record etc.

Default button

Default buttons can be used multiple times per screen and are used to draw focus to common actions which don't significantly impact the user experience.

Semantic buttons

Occasionally, if an action is more significant, essential or impactful, one of three pronounced buttons can be used.

Sizes

There are two sizes available by adding a size1 or size2 class. Size 2 is the default.

Button with data

You can add an icon to support the button text and provide visual reinforcement. For best results, we recommend Feather icons. The icon is wrapped in a div to provide the necessary margin.

Or you can put an icon on the right.

If an icon is sufficient on its own, you can remove the button text.

Or you may need an icon with a chevron and no text.

In some contexts, it can be helpful to add a counter to a button – e.g. follower count, vote count etc. Just separate the counter from the button text with a space.

Here's an example with an icon, text and a counter.

Active state

Each of the four button styles share an active state, which can be applied by adding an is-active class. Example use cases might include signifying that a user is being followed, or that a friend request has been sent.

Disabled state

If an action is usually possible but is currently disallowed for some reason, you can apply a disabled state using the is-disabled class. As shown below, different markup is required depending on the html tag.

Loading state

While in a waiting state, you could swap out the button text for a loading indicator, then swap it back in when the action is complete.

Button group

There are 3 modifier classes, left, middle and right, for grouping buttons in a row or for grouping buttons with other components. Wrap the buttons in a Flexbox component to place them inline. Semantic buttons should not be grouped. One common use case is pagination.

Tags

A variety of html tags can be used to implement a Button, each with their own unique purpose.
